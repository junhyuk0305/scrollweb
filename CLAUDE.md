## 1. 이 프로젝트가 무엇인가

**스크롤트리거 기반 프리미엄 웹사이트를 외주 제작·판매하는 사업.**
핵심 자산은 *직접 코딩한 고퀄 인터랙티브 **핵심 데모 라인업(5종 + 업종별 변형)***이고, 고객에게는
"예시 데모를 기반으로 당신 업종에 맞춰 발전시켜 제작"하는 방식으로 판매한다.

판매 채널: 크몽(상품형) → 위시켓(프로젝트형) 2단계 전략. 비즈니스/계약/세무
지식은 별도 정리되어 있으며 코드 작업과 분리해서 다룬다.

## 2. 상품의 3대 차별점 (모든 산출물이 이걸 증명해야 함)

1. **빠른 납기** — 기본 3일, 기능 포함 1주일. → 매번 처음부터 짜지 말 것. 공통
   레이어(아래 4번)를 재사용해 속도를 낸다.
2. **스크롤트리거 인터랙티브 디자인** — 평범한 웹이 아니라 "애플·토스급 움직임".
   정지 화면이 아니라 *스크롤에 반응하는 경험*이 핵심 가치다.
3. **예시 기반 최소 보장** — 핵심 데모를 먼저 보여주고, 그 퀄리티를 하한선으로
   약속한다. 신규 작업은 항상 가장 가까운 데모에서 출발해 변형·고도화한다.

## 4. 공통 기술 스택 (매번 새로 고르지 말 것)

`_catalog/00_scroll.md` 0번 섹션이 정본. 요약:

| 레이어 | 라이브러리 | 비고 |
|---|---|---|
| 부드러운 스크롤 | **Lenis** | scrub 체감을 결정. 사실상 필수 |
| 스크롤 트리거 | **GSAP + ScrollTrigger** | pin/scrub/진입 전부 담당. 산업 표준 |
| 텍스트 분리 | **SplitType**(무료) | 글자·단어·줄 단위 타이포 효과 |
| 네이티브 대안 | CSS `animation-timeline` | 단순 진입 페이드, 폴백 동반 |

Lenis ↔ ScrollTrigger 동기화 부트스트랩과 효과별 코드는 `_catalog/00_scroll.md`에 있으니
**복붙해서 재사용**한다. 새 의존성 추가는 꼭 필요할 때만.

## 5. 템플릿/페이지 제작 규칙

신규 템플릿을 만들거나 고객 사이트를 제작할 때 따른다:

1. **출발점 고정** — 항상 `docs/21.md`에서 가장 가까운 데모를 base로 잡고 시작한다.
   (예: 펜션 → Reverie Stay, 시공 → Atelier Build, 세무 → Trust Partners)
1-b. **업종 리서치 반영** — `docs/저단가-웹외주-통합보고서.md`에서 해당 업종의 필수 기능·신뢰 신호·
   규제·**스크롤 강도**를 확인하고 적용한다. 스크롤 효과는 업종별로 강약이 다르다:
   B2B/의료/인력/물류 = 절제, F&B/뷰티/숙박/크리에이터 = 적극. 화려함이 신뢰를
   해치는 업종에서는 모션을 낮추고 신뢰 신호·기능 폼을 우선한다.
2. **반응형 필수** — 모바일/태블릿/데스크톱. 모바일에서 인터랙션이 깨지면 안 됨.
3. **접근성** — `prefers-reduced-motion: reduce` 사용자는 scrub/패럴랙스를 차단하고
   즉시 표시되는 폴백을 준다. (가이드의 공통 원칙 준수)
4. **성능** — 무거운 효과(이미지 시퀀스·3D·파티클)는 남발 금지. 한 페이지에
   "주인공" 효과 1~2개 + 보조 페이드 정도. 60fps 유지가 기준.
5. **SEO 기본기** — 메타/OG 태그, 시맨틱 마크업, 이미지 alt. 인터랙션 때문에
   콘텐츠가 크롤러에 안 보이면 안 된다.
6. **CTA** — 모든 상업용 페이지는 고정 CTA(문의/예약) 또는 명확한 전환 지점을 둔다.

## 6. 작업 컨벤션

- **언어**: 문서·주석·커밋 메시지는 한국어. 코드 식별자는 영어.
- **포맷**: 기획 문서는 마크다운.
- **데모 구조(중요)**: 데모는 **원페이지 지양 → 멀티페이지 + 데모당 폴더 + html/css/js 분리**로
  만든다. 폴더 예: `demo-N-슬러그/` 아래 페이지별 `*.html` + `css/styles.css` + `js/app.js`(또는
  `assets/`). 페이지는 홈·소개·서비스·정보·갤러리·문의 등 업종에 맞게 분할.
- **업종당 5 변형**: 한 업종은 5개 변형으로 제작 — 표준 스크롤 강도 3 + 상향 1 + 하향 1.
  변형 폴더는 `<데모명>/v1-슬러그/ … v5-슬러그/`. (자세한 규칙은 `docs/21.md`)
- **파일 네이밍**: 기획 문서는 `숫자_주제.md` 패턴 유지.
- **이 폴더는 git 저장소가 아니다.** 버전 관리가 필요하면 먼저 사용자에게 확인.

## 7. 하지 말 것

- 핵심 데모 라인업을 무시하고 처음부터 임의 디자인으로 만들기 → 항상 base를 고른다.
- 공통 스택을 두고 매 페이지마다 다른 애니메이션 라이브러리 도입.
- 화려함을 위해 가독성·성능·접근성·SEO를 희생.
- 비즈니스/계약/세무 내용을 코드 산출물에 섞기 (관심사 분리).

# 내부 tier 퀄리티 기준 (개발용 · 비공개)

> **목적**: 30/50/80 세 상품의 "보이는 퀄리티 하한선"을 못박아, 데모·납품물이 가격에 맞는 체감을 항상 넘기게 한다.
> 고객 노출 문구는 [상품구성_스크롤홈페이지.md](상품구성_스크롤홈페이지.md), 파일구조·반응형 계약은 [제작표준_파일구조와기획.md](제작표준_파일구조와기획.md) 참조. 이 문서는 **그 사이의 "퀄리티 레버"만** 다룬다.

## 0. 대원칙

- **모든 요금제에 적극적 애니메이션.** 모션 유무로 tier를 가르지 않는다. **30만부터 "요소요소가 다 움직인다."**
- **30만의 시작점 = 레퍼런스 3종 수준** (이게 하한선, 더 낮으면 안 됨):
  - **`portfolio/works/clinic-v2-luxury.html`** — 효과 자체는 평범해도 **모든 요소에 진입 모션**(밀도).
  - **`portfolio/works/demo-7-mode.html`** — 흔치 않은 구성(축소락 히어로·키네틱 마퀴·가로 스크롤·단어 리빌·전 이미지 패럴랙스).
  - **`portfolio/works/demo-6-camping/index.html`** — 멀티페이지 퀄리티 + **풍부한 실제 텍스트**.
- **효과는 카탈로그에서 적극 사용**: [00_scroll.md](_catalog/00_scroll.md) 55종(코어30+시그니처16+확장9) + [00_scroll.html](_catalog/00_scroll.html). 매번 새로 짜지 말고 엔진(`js/app.js`)의 데이터 훅으로 호출.
- **tier는 "효과 *밀도/난이도* + 페이지·nav 구조 + 기능"으로 가른다.** 30도 화려하되, 50·80은 더 많은 시그니처(pin/시퀀스)·멀티페이지·기능을 얹는다.
- **공통 불변(전 tier)**: 반응형 6뷰포트 무결 · `prefers-reduced-motion` 폴백 · CDN 실패 폴백 · 고정 CTA · SEO/OG · 가로 스크롤 0 · 성능 예산(pin ≤3, blur ≤5).

## 1. tier별 하한선 (이 표가 정본)

| 레버 | 30 BASIC | 50 STANDARD | 80 PRO |
|---|---|---|---|
| **구조** | 원페이지 (nav 없음, 로고+CTA만) | 원페이지~3p · **sticky 메뉴 nav** | 멀티페이지감(≤5p) · **풀 메뉴 nav** + 페이지 전환 연출 |
| **nav** | ✕ (스크롤만) | **앵커 메뉴**(섹션 점프 + active 하이라이트 + 모바일 햄버거, *demo-5 식*) | 앵커/멀티 메뉴 + scrolled 축소 + 진행 인디케이터 |
| **섹션 수** | 5~6 | 8~10 | 12~15 |
| **진입 모션(필수)** | 페이드업 + **stagger** + **시차(parallax)** + **히어로 줌** + **숫자 카운트업** | 좌측 전부 + **시그니처 1종**(가로 스크롤 or 핀) + 후기 슬라이드 | 좌측 전부 + **맞춤 2종**(이미지 시퀀스/마스크 리빌) + **프리로더** + **배경색 모핑** |
| **"주인공" 효과** | 보조 효과 충분히 (밋밋함 금지) | 주인공 1 + 보조 | 주인공 2 + 보조 풍부 |
| **기능** | 전화·카톡 고정 CTA | + 문의폼(엑셀·이메일) + 지도 + 후기 | + 이메일 알림 + 운영가이드 |
| **성능** | Lighthouse 85+ | 90+ | 90+ (이미지 최적화·프리로드) |

### 30 BASIC 하한선 (★ 이번 상향 핵심)
기존 30이 "fade-up만"이라 밋밋했음 → **아래는 30에서도 무조건 들어간다**:
1. 히어로 **줌/시차**(약하게라도)
2. 섹션 이미지 **parallax**(`data-speed`)
3. **숫자 카운트업** 1개 섹션
4. **stagger** 카드 진입
5. 고정 CTA(전화·카톡)
→ 즉 **지금까지의 "50 빼기 시그니처/폼/지도" = 새 30**. 효과 밀도는 50에 근접, 기능·nav만 뺀다.

### 50 STANDARD
- **반드시 메뉴 nav**(섹션 이동되는 "진짜 홈페이지" 느낌). active 하이라이트 + 모바일 햄버거.
- 시그니처 1종 = 가로 스크롤 **또는** 핀 섹션.
- 문의폼 + 지도 + 후기 슬라이드.

### 80 PRO
- 메뉴 nav 풀버전(+ scrolled 축소, 스크롤 진행 인디케이터, 섹션 간 전환 연출).
- 맞춤 2종(이미지 시퀀스 + 마스크 리빌) + 프리로더 + 배경 모핑.
- 운영가이드/이메일 알림.
- *주: 실제 납품 PRO는 진짜 멀티페이지(별도 html)도 가능. 데모는 "메뉴 이동 체감"을 앵커 nav로 충족.*

## 2. 효과 팔레트 → tier 매핑 (data-anim / data-* 훅)

| 훅 | 효과 | 30 | 50 | 80 |
|---|---|:--:|:--:|:--:|
| `data-anim="fade-up"` | 진입 페이드업 | ● | ● | ● |
| `data-anim="stagger"` | 자식 순차 진입 | ● | ● | ● |
| `data-speed` | 시차 parallax | ● | ● | ● |
| 히어로 줌/시차 | hero scale+y | ● | ● | ● |
| `data-count` | 숫자 카운트업 | ● | ● | ● |
| `.switcher`/nav | 메뉴 이동 | ✕ | ● | ● |
| `.hscroll` | 가로 스크롤 핀 | ✕ | ● | ● |
| `data-anim="reveal-mask"` | 마스크 리빌 | ✕ | △ | ● |
| `.pin-seq` | 이미지 시퀀스 | ✕ | ✕ | ● |
| `.preloader` | 프리로더 | ✕ | ✕ | ● |
| `data-bg` | 배경색 모핑 | ✕ | ✕ | ● |

> 엔진(`js/app.js`)은 `body[data-tier]`로 강도를 자동 조절한다. parallax 강도 `pFactor`: 30=.3 / 50=.6 / 80=1.

## 3. 포트폴리오 쇼케이스(index) 기준

- index는 **단일 업종이 아니라 여러 직종 예시를 카드로 전시**하는 포트폴리오 허브.
- 카드: 업종 라벨 + 대표 이미지 + 한 줄 + 태그(쓰인 효과). **호버/스크롤 진입 애니** 필수.
- 대표 카드(숙박=윤슬)는 **30/50/80 가격별 비교**로 연결. 나머지는 각 업종 데모로 연결.
- 목적: "이 사람 이만큼 만든다"를 3초에 증명 → 크몽 상세/위시켓 제안서에 그대로 사용.

## 4. 납품 전 셀프 체크 (tier 공통)

- [ ] 6뷰포트(320·375·414·768·1024·1440) 깨짐 0 / 가로 스크롤 0
- [ ] reduced-motion: 모션 꺼도 콘텐츠 100% 노출
- [ ] CDN 차단 시에도 콘텐츠 노출(폴백)
- [ ] 고정 CTA 모바일 노출 · 터치 타겟 ≥44px
- [ ] 30이 "밋밋"하지 않은가? (§1.3 5개 다 들어갔나)
- [ ] 50/80 nav 메뉴 이동·active 정상
- [ ] Lighthouse 모바일 성능·접근성 90+ (30은 85+)

# 크몽 메인·상세 이미지 제작 기획서

> 크몽 상품 "홈페이지 제작(반응형·인터랙티브)"의 **메인 이미지 1장 + 상세 이미지 9장** 제작 기획.
> 와디즈 펀딩 상세페이지식 **스토리 아크**로 구성. 모든 카피는 크몽 이미지 금지어 검수를 통과한 안.
> 관련: [크몽등록_상세페이지원고.md](크몽등록_상세페이지원고.md) · [21.md](21.md) · [상품구성_스크롤홈페이지.md](상품구성_스크롤홈페이지.md)
> 제작 방식: **HTML로 1슬라이드=1섹션 렌더 → 정확한 px 폭으로 스크린샷(PNG)**.

## 0. 전제 — 반드시 지킬 크몽 규칙 (이미지 안 텍스트 기준)

이미지 **안에 넣으면 안 되는 것** (본문 텍스트 필드와 무관, 오직 이미지 픽셀 안):

| 분류 | 금지 | 우리 카피에서의 대체 |
|---|---|---|
| 외부 연락처 | 전화·이메일·SNS·URL·QR | 이미지엔 일절 안 넣음. 데모 링크는 **본문 텍스트**에만 |
| 과장광고 | 최저가·이벤트·매출보장·환불보장·후기5점·100%·별5개 | 사용 안 함 |
| 금액 표현 | 무료·할인·저렴·가성비·0원·"00원에 가능"·부담없는 | "운영 부담을 낮춘 구조" / 가격 숫자는 이미지에 안 넣음(크몽 가격위젯이 담당) |
| 검증불가 | 1위·베스트·No.1·최초·유일·무한·누적 의뢰수/판매액/매출액 | "직접 코딩한 라이브 데모" 같은 **사실 기반** 표현만 |
| 크몽 혼동 | 찜 하트·어워즈/프라임 뱃지·크몽 로고/등급 | 사용 안 함 |
| 타 저작물 | 타인 로고·공공기관 로고·워터마크 이미지 | GSAP 등은 **로고 대신 텍스트**로 표기 |

**추가 주의 (우리 상황 특화):**
- **데모의 picsum/스톡 사진** → 상세 이미지에 크게 박을 땐 AI 생성 이미지·CC0·자체 제작으로 교체하거나, UI 프레임 위주로 보여주고 사진은 부차적으로. (스톡 원본이 그대로 식별되면 "타 저작물" 리스크)
- **데모 속 가상 수치**(누적 투숙객·평점 별점 등)가 화면에 크게 잡히지 않게 크롭. → "내 서비스 실적/후기"로 오해될 소지 차단.
- 데모는 **클라이언트 없는 자체 제작 샘플**이므로 "서비스 샘플 이미지"로 등록 가능(동의 문제 없음). 이 점이 우리 강점.

## 1. 공통 디자인 시스템 (9장 통일)

포트폴리오 사이트(`portfolio/`)와 동일 아이덴티티로 통일 → 일관성 = 신뢰.

| 항목 | 값 |
|---|---|
| 배경 | `--bg #0c0d10` (다크). 흰 크몽 배경에서 자동으로 분리됨 → 흰배경 구분 규칙 동시 해결 |
| 보조 배경 | `--bg-soft #13151a`, 카드 `rgba(255,255,255,.04)` |
| 텍스트 | `--ink #f4f5f7` / 보조 `--ink-dim #a7adba` / 흐림 `--ink-faint #6b7280` |
| 포인트 | `--accent #5b8cff` (블루). CTA·강조·모션 라인에만 절제 사용 |
| 라인 | `--line rgba(255,255,255,.10)` |
| 제목 폰트 | **Noto Serif KR** (700) — 헤드라인·세리프 무드 |
| 본문 폰트 | **Pretendard** — 설명·라벨·표 |
| 브랜드 마크 | `SCROLL**WORKS**` (좌상단 작게, 8장 공통) |
| 모션 표현 | 정지 이미지이므로 **모션 블러 잔상선·반복 고스트·방향 화살표·스크롤 진행바**로 "움직임"을 시각화 |

### 캔버스 규격
- **메인**: `652 × 488px` 고정 (크몽 필수 규격). 썸네일이므로 **카피 최소·대형 타이포**.
- **상세**: **폭 1000px 고정**(크몽 허용 652~2000 안, 레티나에서 선명). 높이는 슬라이드별 1200~2600px(≤3000 준수).
- 슬라이드 간 **상하 80~120px 여백**으로 시각적 구분(크몽 권장: 이미지 사이 여백).
- 등록 순서 = 노출 순서. **D1→D9 논리 순서대로** 업로드.

## 2. 메인 이미지 (1장) — 652×488

**목표:** 검색 결과 썸네일 그리드에서 0.5초 안에 "스크롤에 반응하는 프리미엄 홈페이지 제작"을 각인. 작게 보여도 읽혀야 함.

**레이아웃 (가로형 4:3):**
```
┌────────────────────────────────────────────┐
│ SCROLLWORKS                          (작게)  │
│                                              │
│   스크롤에 반응하는            ┌───────────┐ │
│   홈페이지              ←모션선 │  데스크톱  │ │
│                                │  목업 화면 │ │
│   반응형 · 인터랙티브 · 빠른 납기│  (스크롤   │ │
│                                │   잔상)   │ │
│   [GSAP] [반응형] [모바일]      └───┐ ┌────┘ │
│                                  모바일 목업  │
└────────────────────────────────────────────┘
```
- **헤드라인(세리프, 초대형):** `스크롤에 반응하는 / 홈페이지` (2줄, 줄당 ≤8자)
- **서브(Pretendard):** `반응형 · 인터랙티브 · 빠른 납기`
- **칩 3개:** `GSAP` `반응형` `모바일` (테두리 outline, accent 1개만 채움)
- **우측 비주얼:** 데스크톱 브라우저 목업 + 겹친 모바일 목업. 화면 안엔 데모(예: 펜션/캠핑 다크 히어로)를 넣되, **스크롤 모션 잔상선**을 accent로 1~2줄 그어 "움직임" 암시.
- 금지어 0. 가격·연락처·과장 없음.

> 대안 헤드라인: `스크롤트리거 / 인터랙티브 웹` / `움직이는 / 홈페이지 제작`. → A안(스크롤에 반응하는 홈페이지) 권장.

## 3. 상세 이미지 9장 — 와디즈식 스토리 아크

흐름: **후킹 → 공감(문제) → 해결 선언 → 차별점 3개 → 구성/범위 → 신뢰(프로세스·보장) → 마무리 CTA**.

---

### D1 · 표지 / 핵심 가치 (Hook)  — 높이 ~1600
- **카피(대형 세리프):** `스크롤을 내리면 / 살아나는 홈페이지`
- **서브:** `정지된 화면이 아니라, 스크롤에 반응해 움직이는 인터랙티브 홈페이지를 만듭니다.`
- **비주얼:** 풀블리드 다크 히어로 + 대표 디바이스 1대(스크롤 진행바·모션 잔상). 하단에 ↓ 스크롤 유도 인디케이터.
- 역할: 톤 세팅 + "왜 정지된 웹이 아닌가" 한 방.

### D2 · 문제 제기 / 공감 (Problem)  — 높이 ~1800
- **카피:** `혹시, 이런 홈페이지인가요?`
- **공감 카드 4개(체크/엑스 아이콘):**
  1. `몇 년째 그대로인 오래된 디자인`
  2. `모바일에서 레이아웃이 깨진다`
  3. `네이버 플레이스·블로그에만 의존`
  4. `어디서 본 듯한 평범한 템플릿`
- **비주얼:** 흐릿한 "구식 사이트" 와이어프레임을 회색 톤으로, 위에 X 마커. 와디즈식 "공감 → 전환" 진입.

### D3 · 해결 선언 (Solution)  — 높이 ~1500
- **카피:** `그래서, 이렇게 만듭니다` → `스크롤에 반응하는 / 프리미엄 인터랙티브 홈페이지`
- **3대 가치 한 줄 요약(아이콘 3):** `① 스크롤 인터랙션  ② 반응형·모바일  ③ 빠른 납기`
- **비주얼:** D2의 회색 구식화면이 컬러풀한 다크 프리미엄 화면으로 "전환"되는 비포→애프터 split. accent 강조.

### D4 · 차별점 ① 스크롤 인터랙션 (Depth)  — 높이 ~2200
- **카피:** `단순 템플릿 수정이 아닌, / 스크롤 효과를 직접 코딩합니다`
- **설명:** `GSAP(ScrollTrigger) 기반으로 페이드인·핀·패럴랙스·타이포 모션 등 업종과 목적에 맞는 효과를 조합합니다.`
- **효과 4종 다이어그램(정지로 모션 표현):**
  - `페이드 인 / 라이즈` (아래→위 잔상)
  - `패럴랙스` (레이어 깊이 화살표)
  - `핀 고정 + 스크럽` (구간 고정 표시)
  - `타이포 분할` (글자 단위 분해)
- **배지:** `보유 스크롤 효과 55종 조합` (수치는 보유 효과 개수 = 사실, 검증불가 표현 아님 → 허용). "누적 판매" 류 아님.
- 역할: 우리 핵심 기술력 증명.

### D5 · 차별점 ② 업종별 라이브 데모 라인업 (Proof — 가장 강함)  — 높이 ~2600
- **카피:** `말보다 결과로 — / 업종별 라이브 데모를 직접 확인하세요`
- **설명:** `직접 코딩한 자체 제작 데모입니다. 가장 가까운 사례에서 출발해 고객님 업종에 맞춰 발전시킵니다.`
- **데모 그리드(디바이스 썸네일 6개 + 라벨):**
  펜션·풀빌라 / 인테리어·시공 / 카페·F&B / 의원·한의원 / 세무·노무 / 캠핑·글램핑
- **비주얼:** 각 데모 다크 히어로를 디바이스 프레임에 카드로. **사진은 AI/CC0로 교체 또는 UI 프레임 위주**(스톡 원본 노출 금지), 데모 속 수치/별점은 크롭 제외.
- **주의:** 이미지에 데모 **URL 금지** → "라이브 데모 제공"만 표기. 실제 링크는 **본문 텍스트**에.
- 역할: "예시 기반 최소 보장"의 핵심 증거.

### D6 · 차별점 ③ 반응형·모바일 + 성능·접근성  — 높이 ~1900
- **카피:** `PC에서 본 그 움직임, / 모바일에서도 그대로`
- **비주얼:** 동일 화면의 PC/태블릿/모바일 3-디바이스 정렬.
- **포인트 칩:** `모바일·태블릿·PC 반응형` · `스크롤 효과 모바일 최적화` · `60fps 목표` · `저사양·접근성 환경 자동 경량화(prefers-reduced-motion)`
- 역할: "모바일에서 깨진다"는 D2 불안을 직접 해소.

### D7 · 제공 범위 & 패키지 구성 (Scope)  — 높이 ~2400
- **카피:** `필요한 만큼, 패키지로 선택하세요`
- **3티어 구성 비교(가격 숫자 없이 — 가격은 크몽 가격위젯이 담당):**

| | 기본형 | 표준형 (추천) | 풀 인터랙티브 |
|---|---|---|---|
| 구성 | 원페이지(4~6섹션) | 8~10섹션 / 메인+서브 | 최대 5p (12~15섹션) |
| 스크롤 인터랙션 | 기본 | 시그니처 1종 | 맞춤 2종 |
| 반응형 | ● | ● | ● |
| 문의 폼 | – | ● | ● |
| 후기 슬라이드·지도 | – | ● | ● |
| 이미지 시퀀스·마스크·로딩 | – | – | ● |

- **공통 제공 체크리스트:** `반응형` `기본 SEO(메타·OG)` `고정 CTA(상담/전화/카톡 버튼)` `소스 코드 + 주석 납품`
- **주의:** 이미지에 **가격 원 숫자·수정횟수 미표기**(원고와 정합성 이슈 미해결 + 금액 금지어 회피). 패키지 "내용"만.

### D8 · 제작 프로세스 + 납품·소유권·보장 (Trust)  — 높이 ~2200
- **카피:** `요청부터 내 소유의 사이트까지`
- **5단계 타임라인:** `요청서 확인 → 범위·방향 확정 → 제작 → 시안·수정 반영 → 납품·배포 인계`
- **신뢰 블록 3카드:**
  1. `소스·계정·도메인 모두 고객님 소유` (특정 업체에 묶이지 않음)
  2. `GitHub Pages 정적 호스팅으로 운영 부담을 낮춘 구조 + 자동 HTTPS` ← "무료/0원" 금지어 회피한 표현
  3. `납품 후 1개월 하자보수 포함`
- 역할: 와디즈식 "리워드 신뢰" = 결제 직전 불안 제거.

### D9 · 마무리 CTA + FAQ 압축 (Closing)  — 높이 ~1800
- **카피:** `업종만 알려주시면, / 가장 가까운 데모에서 시작합니다`
- **핵심 FAQ 4개 압축:**
  - `호스팅 — 고객님 계정 정적 호스팅으로 운영 부담을 낮춤`
  - `소유권 — 계정·소스·도메인 전부 고객님 소유`
  - `문의 수신 — 구글시트 자동 적재 또는 메일 도착(표준·프리미엄)`
  - `규제 업종 — 의료·법무·학원 금지표현 검수 후 제작`
- **클로징:** `구매·문의로 시작하세요` (크몽 내 행동 유도, **외부 연락처 없이**) + `SCROLLWORKS` 사인오프.

## 4. 제작 워크플로 (HTML → PNG)

1. **1 HTML = 1 슬라이드** (또는 한 파일에 9섹션 + 슬라이드별 `width:1000px; height:Npx` 컨테이너).
2. 공통 `image.css`로 컬러·폰트·디바이스 목업·모션 잔상 유틸 정의 → 9장 톤 통일.
3. 메인은 별도 `main-652x488.html`.
4. **Playwright(webapp-testing 스킬)로 각 컨테이너를 정확한 px 뷰포트에서 `screenshot` → PNG.** 레티나 선명도 위해 `deviceScaleFactor: 2`로 찍고 1배 규격으로 다운스케일 권장.
5. 사진 자리: 사용자 제공 AI 이미지/CC0로 교체 후 재렌더.

**산출 파일 제안:**
```
docs/images/
  main-652x488.html
  detail-01-hook.html … detail-09-closing.html
  image.css
  out/  (스크린샷 PNG 9 + 메인 1)
```

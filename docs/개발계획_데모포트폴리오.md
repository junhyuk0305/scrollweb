# 개발 계획 — 코어 타겟 데모 → 정적 포트폴리오 → 크몽 등록

> **스택: Plain HTML/CSS/JS + GSAP/Lenis/ScrollTrigger(CDN) + 무료 호스팅(고객 계정, 제공처 검토중) + 폼은 구글시트/이메일.** Next.js·백엔드 미사용(저관리·빠른회전).

> 기준: [상품구성_스크롤홈페이지.md](상품구성_스크롤홈페이지.md) (저가 30~80만 · 오버스펙 스크롤 애니 · 무료 호스팅 고객계정 · 저관리).
> 목적: 이미 만들어 둔 데모 자산을 **판매용 라이브 포트폴리오**로 전환하고, BASIC/STANDARD/PRO를 실물로 증명한 뒤 크몽에 올린다.

## 1. 코어 타겟 선정 (우선순위)

선정 기준: ①오버스펙 스크롤이 잘 먹히는 비주얼 업종 ②소상공인 물량·인텐트 ③저관리(예약·결제 외부링크화 가능) ④데모 보유.

### 1차 코어 (먼저 판다)
| 순위 | 타겟 | 근거 | 대표 데모 | 증명 tier |
|---|---|---|---|---|
| 1 | **인테리어·건설 시공** | 세그B(사이트 없음), 시공 포트폴리오+견적폼 인텐트 높음 | demo-2-build | STANDARD |
| 2 | **F&B 다이닝·카페** | 화려한 스크롤 최적, 브랜딩 수요, 빠른 회전 | demo-3-savor | BASIC |
| 3 | **숙박·풀빌라·캠핑** | 감성 비주얼, 예약은 외부링크=저관리 | demo-1-stay / demo-6-camping | BASIC / PRO |
| 4 | **의료(한의원·피부과)** | 규제 대응 차별화, 멀티페이지 수요 | demo-4-care / clinic | PRO |

### 2차 확장
- 전문직 B2B(세무·노무) — demo-5-trust (STANDARD)
- 인력·용역(단가표+견적폼) — 데모 신규 필요

### 데모 → tier → 증명 매핑
- **BASIC(4~6섹션)**: savor, stay 원페이지
- **STANDARD(8~10섹션+시그니처애니+문의폼)**: build, trust
  - ⚠️ 현재 build는 7섹션 → STANDARD 하한(8~10섹션) 맞추려면 1~3섹션 보강 필요.
- **PRO(12~15섹션+멀티페이지+맞춤애니)**: care(멀티), camping(멀티)
  - ⚠️ clinic은 vanilla 스택이라 PRO 증명에서 제외 → GSAP 통일(2차) 후 편입.

## 2. 순차 개발 계획 (Phase별 · 검증 기준 포함)

> 원칙: 데모를 하나씩 늘리지 말고 **공통 엔진 먼저**(빠른 회전의 핵심). 각 Phase는 "검증" 통과해야 다음으로.

### Phase 0 — 공통 엔진 (정적 셸 + 애니 스니펫 라이브러리)
1. **공통 셸**(헤더·고정 CTA바·푸터·SEO/OG 메타·리셋 CSS·GSAP/Lenis CDN 로더) 1벌 + CSS 변수 토큰(색/폰트/간격) → verify: 빈 셸이 그대로 배포되고 반응형 정상
2. **애니 스니펫 라이브러리**: 55종을 `data-anim="..."` 속성으로 호출하는 재사용 JS 1파일 + `prefers-reduced-motion` 폴백 → verify: 모션 끔 환경에서 콘텐츠 정상 노출
3. **BASIC 기본 애니 세트** 우선(히어로 트랜지션1·페이드업·시차·숫자 카운트업·핀1) → verify: 쇼케이스 1페이지에서 5종 동작
4. **GitHub Pages 배포 파이프라인** 1회 관통(고객 계정 repo → Pages → 커스텀 도메인) → verify: 커밋→배포→라이브 URL
- ✅ **Phase 통과 기준**: 셸+스니펫으로 만든 데모 1종이 GitHub Pages 배포→스크롤·폴백 정상, Lighthouse 90+
- *호스팅 = **GitHub Pages 확정**(무료·CDN·상업OK·커스텀도메인·무료 HTTPS). 폼은 구글시트 또는 이메일.*
- *비고: 빌드 도구 없이 순수 HTML. 섹션은 스니펫 복붙 + `data-anim` 지정. 물량 늘어 복붙이 아프면 그때 Astro 검토(상품구성 §스택).*

### Phase 1 — 코어 BASIC/STANDARD 데모 3종 라이브 (포트폴리오 URL 확보)
1. **savor**(BASIC) 셸 포팅 → 라이브
2. **stay**(BASIC) 셸 포팅 → 라이브
3. **build**(STANDARD, 견적폼은 Phase 3 전까지 외부링크) → 라이브
- ✅ 통과: 3개 라이브 URL, 모바일 정상, 각 Lighthouse 90+ → **크몽 상세페이지에 붙일 데모 3종 확보**

### Phase 2 — 시그니처 애니 4종 (STANDARD 무기)
1. 가로 스크롤 / 이미지 시퀀스 / SVG 선 그리기 / 카드 스태킹 컴포넌트화
2. build·trust에 시그니처 1종씩 적용
- ✅ 통과: 4종 애니 독립 동작 + 저사양/모바일 폴백 확인

### Phase 3 — 문의폼 모듈 (구글시트 / 이메일)
1. 재사용 **문의폼 컴포넌트** → ①Google Apps Script 웹훅으로 **구글시트 자동 적재**, 또는 ②무료 폼서비스(web3forms 등)로 **이메일 수신** (허니팟 스팸 차단)
2. 구글시트는 **엑셀(xlsx) 다운로드** 그대로 가능 — 별도 DB·관리자·서버 없음
3. STANDARD/PRO 데모에 탑재 (build 견적폼 실연동)
- ✅ 통과: 제출→구글시트 적재 & 이메일 도착 E2E, 백엔드 0

### Phase 4 — PRO 멀티페이지 데모 2종
1. **care**(의료 멀티 6p) 포팅 + 맞춤 애니 1종 + **의료광고 규제 카피** 점검(§규제)
2. **camping**(멀티 6p) 포팅 + 맞춤 애니 1종
- ✅ 통과: 멀티페이지 라우팅·공유태그 정상, 규제 금지표현 0건, Lighthouse 90+

### Phase 5 — 크몽 패키징·등록
1. 상세페이지 카피(상품구성 §9) + **데모 URL 6종** + 가격표 + 규제 배지
2. BASIC/STANDARD/PRO 3패키지 등록, **견적 역산**(÷0.87, VAT 별도) 가격 세팅
3. 썸네일·대표 GIF(스크롤 시연)
- ✅ 통과: 리스팅 게시, 모든 데모 링크 클릭 동작, 문의 1건 수신 테스트

### Phase 6 — 확장(2차)
- 인력·용역 데모 신규 제작(단가표+견적폼)
- clinic 5종 GSAP로 스택 통일(2차)
- 후기·실주문 포트폴리오 적재 → 상위 노출

## 3. 의존성·리스크

- **Phase 0가 전체 속도를 결정.** 공통 셸·애니 스니펫을 잘 짜면 Phase 1·4가 "스니펫 복붙+data-anim" 수준 → "빠른 회전" 달성.
- 기존 demo-1~6은 이미 GSAP plain HTML → **공통 셸/토큰에 맞춰 정리 + data-anim 통일**만 하면 됨(재작성 아님).
- clinic(vanilla)은 급하지 않음 → **2차로 미룸**(스니펫 라이브러리로 교체).
- 무료 호스팅: **GitHub Pages 확정**(무료·CDN·상업OK·커스텀 도메인·무료 HTTPS, 정적 전용=우리 스택과 일치). 차순위 대안 Cloudflare Pages. 폼은 구글시트(Apps Script) 또는 이메일.
- 규제 업종(의료) 데모는 **카피 검수 없이 공개 금지**(§규제 리스크).

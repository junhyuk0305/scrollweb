## 4. 아직 손도 안 댄 것 ❌ (= 판매를 막고 있는 병목)

1. **포트폴리오 허브 미연결** — `portfolio/index.html`은 가격·문의 랜딩 페이지일 뿐, **데모 카드로 각 작업물을 링크하지 않는다.** ([내부기준 §3](../내부_tier퀄리티기준.md)가 요구하는 "여러 직종 카드 전시 허브"가 아직 없음. 현재 index의 링크는 `#price`, `mailto:` 뿐.)
2. **라이브 배포 0** — 이 폴더는 **git 저장소조차 아니다.** GitHub Pages(확정 호스팅) 미설정 → **공개 URL이 하나도 없다.** 크몽 상세에 붙일 데모 링크가 없음.
3. **공통 엔진(`_engine`) 미생성** — 계획상 Phase 0의 핵심. 현재는 데모마다 개별 `app.js`/`css`. 빠른 회전(3~7일 납기)의 전제가 아직 안 깔림.
4. **문의폼 모듈 없음** — 구글시트/이메일 수집 미구현. index는 `mailto:` 하드코딩(`cristianojun@naver.com`). STANDARD/PRO의 핵심 기능 공백.
5. **yunseul 30/50/80 비교 미구현** — 대표 카드를 "가격별 3종 비교"로 보여줘야 하는데(내부기준 §3) 현재 단일 버전으로 보임.
6. **크몽 미등록** — 썸네일·시연 GIF·패키지 게시 전부 미착수.

   - `portfolio/index.html`에 8개 데모를 업종 카드로 깔고 각 `works/*.html`에 링크.
   - 대표 카드(윤슬)는 30/50/80 비교로 연결.

**② (1~2일) 라이브 배포 — git init → GitHub Pages**
   - 이 폴더를 git 저장소로 만들고(사용자 확인 필요 — CLAUDE.md §6) GitHub Pages로 `portfolio/` 공개.
   - → **공개 URL 확보.** 이게 없으면 크몽 등록 자체가 불가.

**③ (2~3일) 문의폼 모듈 1벌** — Google Apps Script 웹훅 → 구글시트 적재(+이메일). 재사용 컴포넌트로. STANDARD/PRO 데모에 탑재.
